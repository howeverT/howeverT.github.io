    架构篇

    SOLID原则

    单一职责原则

    • 一个类负责一个职责。（高内聚）

    • 如果一个类有多个职责，就应该拆分为多个类。（低耦合）

    开闭原则

    • （类、模块、函数等）应该可以扩展，但不必为扩展而修改。

    • 主要使用面向对象编程语言的抽象机制：抽象类、接口等

    里氏替换原则

    • 引用了基类的功能在使用派生类对象代替基类后应不受影响，且不须知道具体为何种派生类。

    • 满足LSP原则主要以继承为基础，通过抽象和多态实现。

    Ostrich不会飞

    接口隔离原则

    • 客户不应该被强迫去依赖他们不使用的接口。

    • 每一个接口所含方法具有合理的内聚性。

    依赖倒置原则

    • 高层模块不应依赖低层模块

    • 层之间依赖接口。即：高层定义接口，低层实现接口。高层按接口使用低层功能

      实际上是低层依赖高层的接口

    模式

    单例模式

    • 单例模式只能有一个实例。

    • 单例类必须创建自己的唯一实例。

    • 单例类必须向其他对象提供这一实例。

    java
    适配器模式
    • 使用一个适配器类实现客户类要求的接口，同时又继承使用类或者聚集使用类。

    java
    工厂模式
    • 工厂：负责实现创建所有实例的内部逻辑，并提供一个外界调用的方法，创建所需的产品对象。
    • 抽象产品：负责描述产品的公共接口。
    • 具体产品：描述生产的具体产品。
    java
    抽象工厂模式
    • 抽象工厂：描述具体工厂的公共接口
    • 具体工厂：描述具体工厂，创建产品的实例，供外界调用
    • 抽象产品族：描述抽象产品的公共接口
    • 抽象产品：描述具体产品的公共接口
    • 具体产品：具体产品

    数据库篇

    如何设计一个关系型数据库
    • 要考虑存储管理、缓存机制、索引管理、锁管理、权限划分、日志管理、sql解析等
    • 将数据持久化存储到存储（文件系统）中
    为什么使用索引
    • 全表扫描在多行数据时比较慢，避免全表扫描
    • 快速查询数据
    什么信息可以成为索引
    • 主键、唯一键、普通键
    索引的数据结构
    • 二叉查找树，B+树，hash结构
    • 为什么不使用红黑树等，树高很高，检索性能会下降
    • 使用hash就不能使用范围查询，只能满足=，in，无法被用来避免排序操作，不能利用部分索引键查询，还是要表扫描，hash值相等多后性能差
    密集索引和稀疏索引

    • 密集索引：每个文件的每个搜索码对应一个索引值

    • 稀疏索引：只为索引码某些值建立索引项

    mysql的两种存储引擎

    • myISAM：主键、唯一键等，属于稀疏索引。数据和索引分别存在两个文件中

    • InnoDB：有且只有一个密集索引，一个主键被定义，则就是密集索引，没有主键，就是第一个唯一非空索引做密集索引，若都不满足创建隐藏主键。非主键索引存储键位和对应主键值，是2次查找

    sql调优
    • 慢日志定位
    • explain工具分析
    • 修改sql走索引
    联合索引最左匹配原则

    • 设置（a,b）为联合索引

    • where a= and b= 和where a=是走联合索引，where b=就不走联合索引

    myISAM与InnoDB锁方面的区别

    • myISAM默认表级锁，不支持行级锁（差2000以前的，查2000以后的会被锁住，行级锁不会）

    • InnoDB默认行级锁，支持表级锁，加减锁——二级锁

    • 表级锁：多个并发操作表，当前面语句操作表时，后面的语句等待前面的执行完成，自动上锁

    • 读锁（共享锁） lock tables __ read

    • 写锁（排他锁）

    • 不走索引时整张表都会锁住，InnoDB没有用到索引时是用表级锁

    X是排他锁，S是共享锁

    myISAM与InnoDB适用的场景

    乐观锁与悲观锁
    • 悲观锁：数据库的锁机制，排他锁实现，数据库的锁方式，容易死锁
    • 乐观锁：提交时才对数据冲突进行检测，版本号或时间戳
    数据库事务四大特性(ACID)
    • 原子性：事务全部操作全部执行或全部回滚
    • 一致性：事务结束后数据一致
    • 隔离性：一个事务不应该影响另一个事务
    • 持久性：事务执行后是永久性的(InnoDB:写到了一个文件里，打开的时候进行恢复)
    事务并发引发的问题

    InnoDB可重复度隔离级别避免幻读

    • 表象：快照读（非阻塞读），当前读，是最新的，会加锁lock in share mode

    • 内在：next-key锁（行锁+gap锁）

    • gap锁：锁定一个范围，where都命中，只加记录锁，部分命中或都不命中就会加gap锁

    关键语法
    • group by:（同一表）select子句中列名必须是分组列和列函数，列函数对group by子句定义的每个组返回一个结果，where在group by前面
    • having：与group by一起用，having过滤组

    Java篇

    底层JVM

    对Java的理解
    • 平台无关性—一次编译到处运行
    • GC 垃圾回收
    • 语言特性 泛型，反射等
    • 面向对象（继承、多态、封装）
    • 类库
    • 异常处理
    平台无关性的实现
    • 编译成class文件 javac执行，二进制字节码

    JVM如何加载.class文件

    • Java虚拟机：虚拟机将字节码解析成适合本操作系统的二进制码

    Java的反射

    • 对于任何一个类，都知道这个类所有属性方法，对于任意对象都能调用它的方法和属性。（动态获取信息和动态调用对象方法）

    • 反射可以获取private的方法，不嫩获取继承的方法和实现接口的方法

      java
      类从编译到执行的过程

    谈ClassLoader

    • 从系统外部获得Class二进制数据流

    • 所有Class都由ClassLoader进行加载

    • ClassLoader将二进制数据流装在进系统交给虚拟机初始化等

    • ClassLoader的种类

    • CLassLoader的双亲委派机制

    类的加载机制

    • 隐式：new

    • 显式：loadClass、forName（获取到Class之后要用newInstance生成实例，要用反射）

    • 类的装载过程

    forName和loadClass的区别
    • forName是已经完成初始化
    • loadClass是还没进行链接的（只完成了加载）——>Spring的懒加载
    Java的内存模型

    • 内存模型

    • 程序计数器：当前线程执行字节码行号指示器、改变计数器值取下一条、和线程一对一关系（线程私有），对java方法计数，Native方法则为undefined、不会发生内存泄漏

    • 虚拟机栈：java方法执行的内存模型、多个栈帧（局部变量表，操作数栈）

    垃圾回收机制GC

    标记算法

    • 判断是否垃圾：引用计数法（被引用+1、无法循环引用）、可达性分析算法（GC Root到达，没标记的回收）

    • GC Root 的对象

    回收算法
    • 标记清除算法（可达性）：会有碎片化
    • 复制算法：对象面和空闲面，对象在对象面创建，存活的复制到空闲，清楚对象面（对象存活率低–年轻代）
    • 标记整理算法：根集合扫描对存活进行标记，移动存活，将末端回收
    • 分代收集算法：不同生命周期划分不同区域
    GC的分类

    • Minor GC:复制算法

    • Full GC:和Major GC基本一致，比MinorGC慢，但执行频率低。老年代、永久代空间不足、CMSfailed、调用System.gc()时触发

    • 年轻代：Eden区和2个Survivor区

    Java多线程与并发—原理

    synchronized

    • 互斥锁：其他线程等待，同一时间只允许一个线程使用对象锁，原子性。可见性：共享变量可见

    • 锁的是对象，不是代码

    • 异步的后启动的可能先完成

    • 同步代码块是顺序执行的，代码块外面的是异步的

    • 同步方法就是完全同步，完成后才启动

    • 对同一线程，不同线程就是同一个类的不同对象互不干扰，异步的

    • 每个类只有一个类锁

    • 同一对象跟之前的对象锁是一致的

    • 不同实例时类是一样，同步的代码是受类锁制约，同步的

    • 类锁对象锁互不干扰

    synchronized的底层实现原理

    • 对象在内存中布局：对象头、实例数据、对齐填充

    • Java对象头：Mark Word、Class Metadata Address

    • Monitor：java对象自带看不见的锁，堪称同步机制

    • synchronized是重量级锁，要从用户态转到核心态

    • 自旋锁：循环等等锁释放，但不让出CPU

    • 自适应自旋锁：自旋次数比在固定，拥有者状态决定

    • 锁消除：去除不可能存在竞争的锁

    • 锁粗化：扩大加锁范围防止频繁加锁和减锁

    • synchronized4状态：无锁、偏向锁、轻量级锁、重量级锁
